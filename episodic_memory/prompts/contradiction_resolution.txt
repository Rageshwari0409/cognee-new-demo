You are evaluating a potential conflict between an existing fitness memory record and new information from a recent AI coaching conversation.

The existing record is a memory the AI already holds. The new information comes from a more recent conversation. Your task is to decide what to do with the new information: does it correct the existing memory, add to it, or describe something entirely different?

---

EXISTING RECORD:
Episode type: {{existing_episode_type}}
Occurred on: {{existing_occurred_on}}
Outcome: {{existing_outcome}}
Source quotes from original conversation: {{existing_source_quotes}}

---

NEW INCOMING INFORMATION:
Episode type: {{new_episode_type}}
Occurred on: {{new_occurred_on}}
Outcome: {{new_outcome}}
Source quotes from new conversation: {{new_source_quotes}}

---

DECISION RULES

"contradiction" — the user has stated something that directly conflicts with a specific fact already recorded. This means the AI's memory is wrong and needs to be corrected. Classic examples: correcting a number ("actually it was 3km, not 5km"), retracting an outcome ("I said I finished the session but I actually left early"), explicitly contradicting a prior claim. Use action "supersede".

"extension" — the new information adds new detail to the same event without conflicting with anything already recorded. The existing record remains accurate; it is simply incomplete. Classic examples: adding the name of the program to a record that said "started a new program", clarifying which muscle group was trained. Use action "merge".

"new_event" — the new information describes a different event, a different session, or a different topic that does not meaningfully overlap with the existing record. Both the existing record and the new information should exist as separate memories. Use action "create".

EDGE CASES:
- Same topic, different date: almost always "new_event"
- Same date, different detail that does not conflict: "extension"
- User says "actually" or corrects a number or outcome: almost always "contradiction"
- User provides more detail about the same session: "extension" unless the detail conflicts

---

OUTPUT FORMAT

Return a single JSON object. No explanation outside the object.

{
  "decision": "contradiction" | "extension" | "new_event",
  "reasoning": "One or two sentences explaining the decision. Be specific about which fact conflicts, extends, or is unrelated.",
  "action": "supersede" | "merge" | "create"
}

---

Existing record:

  occurred_on: 2026-06-10
  outcome: "Completed 5km run, felt strong throughout"
  source_quotes: ["Did the full 5k, felt really good"]

New incoming:

  occurred_on: 2026-06-10

Expected output:
{
  "decision": "contradiction",
  "reasoning": "The existing memory states the run was completed at 5km. The user's new statement explicitly corrects this to approximately 3km with a specific reason. The existing memory is factually wrong and must be replaced.",
  "action": "supersede"
}
